# 节点颜色语义：红黑树的红 / 黑，与 AI 讲解高亮

## 红黑树里的节点为什么是红色（或黑色）
红黑树（Red-Black Tree）给每个节点着"红"或"黑"两种颜色，是用最少的额外信息维持"近似平衡"的手段。核心规则：
- 每个节点非红即黑；**根节点永远是黑色**。
- 红色节点不能有两个连续的红色子节点（红红不相邻：红色节点的父节点必为黑色）。
- 从任一节点到其所有叶子（NIL）的每条路径上，黑色节点数量相同（黑高相等）。

**一个节点显示为红色，通常是因为插入时先把它染成红色**：新节点一律先设为红色，这样不会破坏"黑高相等"这条约束；只有当它的父节点也是红色（出现红红冲突）时，才通过**重新着色（recolor）+ 旋转**修复。也就是说，"红色"是插入修复过程中的中间着色状态，很多节点在修复完成后会被重新染回黑色。

所以你在可视化里看到某个节点是**红色**，它大概率刚插入、或正处于某次插入 / 删除修复的着色阶段；等它最终稳定后往往会被染回黑色（除非它恰好是允许为红的节点）。

## 红色 ≠ AI 讲解高亮
注意区分两种"颜色"：
- **红黑树自身的红 / 黑**：数据结构属性，由算法维护，代表平衡约束。
- **AI 讲解助教的高亮色**：是另一种颜色（青绿色 `#0F6E56`），它只是**标记"当前讲解正在讨论这个节点"**，和红黑树的红 / 黑没有任何关系。高亮出现 = RAG 认为这个节点的"值"与当前讲解内容相关，纯属视觉提示，不代表该节点在树里是红色。

## 面试怎么答"为什么这个节点是红色"
"红黑树新节点一律先染红，因为红色不增加黑高，最容易满足黑高约束；只有当出现红红冲突时才靠 recolor / 旋转修复，修复后多数节点回到黑色。所以看到红色节点，说明它刚插入、或正处于一次修复的着色阶段——这是算法的中间状态，不是错误。"
